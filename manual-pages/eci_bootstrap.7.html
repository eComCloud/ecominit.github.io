NAME

eci_bootstrap —
early initialisation and setup of eComInit

DESCRIPTION

The ecominit(7) system is modular and multi-process and hence initial startup involves a specific set of tasks to be carried out, during which the system operates with limited functionality. This document describes that bootstrap process.

Environment Setup

ecinit(8) runs first. This handles the specific tasks needed to get a system or session to the point where eci.startd(8), the master restarter, can run. It then launches (and supervises) the master restarter.

Early Manifest Import

eci.startd(8) sets up two pseudo-instances: these are hardcoded instance descriptions built into eci.startd(8) which invoke eci.configd(8) and then, the Manifest Import service.

The Manifest Import service scans a set of standard directories for service manifests. The specific directories scanned varies depending on whether this is a system or a session instance. See ecominit(7), section Layers, for a detailed list.
